一 ：Solr 安装与配置

1.1 解压 tomcat

将tomcat安装包解压到任意目录下

1.2 解压 Solr 安装包

将solr-4.10.3.zip解压，解压后如下：

1.3 将solr.war部署到tomcat

将【solr解压目录/dist】下的solr-4.10.3.war改为solr.war，并拷贝到tomcat/webapp目录下

1.4 将扩展包部署到solr工程下

1.5 1.1.1 创建solrhome索引库目录

拷贝【solr解压目录\example\solr】到d盘（可以是任意不带空格和中文的目录），并改名为solrhome

1.6 在tomcat中关联solrhome目录

方式一：

修改solr工程下的web.xml

    <env-entry>
       <env-entry-name>solr/home</env-entry-name>
       <env-entry-value>D:\solrhome</env-entry-value>
       <env-entry-type>java.lang.String</env-entry-type>
    </env-entry>

方式二：本文使用的是该方式

修改【tomcat解压目录\bin】下catalina.bat文件，在第二行添加如下代码：

set JAVA_OPTS="-Dsolr.solr.home=D:\solrhome"

1.7 重启tomcat并访问solr

在浏览器输入如下地址：

http://localhost:8080/solr

三 ：IKAnalyzer分词器

3.1 概述：

Solr默认的分词器是一元分词器，是针对英文进行分词的，不适合中文，如果希望Solr支持中文分析，需要单独配置中文分词器。而IK分词器就是一款不错的中文分词器。

3.2 IKAnalyzer分词器配置

3.2.1将分词器jar拷贝到Solr工程的lib目录下

将Solr\IK Analyzer 2012FF_hf1】下的IKAnalyzer2012FF_u1.jar拷贝到solr工程目录下

3.2.2将分词器资源拷贝到solr 工程/WEB-INF/classes下

将Solr\IK Analyzer 2012FF_hf1】下的扩展词典、停用词词典、配置文件放到Solr程的 WEB-INF/classes 目录下

3.2.3 在solrhome中配置IK分词器

在自定义Solr分词库目录下的schema.xml中加入如下配置 

<fieldType name="text_ik" class="solr.TextField">
<analyzer class="org.wltea.analyzer.lucene.IKAnalyzer"/>
</fieldType>

四 ：配置服务域

4.1 域简介

l 域（field）相当于数据库的表字段，用于存储用户数据

l 域的常用属性：

Ø name：指定域的名称

Ø type：指定域的类型

Ø indexed：是否索引

Ø stored：是否存储

Ø required：是否必须

Ø multiValued：是否多值

l 域的分类

Ø 一般域：对应数据库中的一般字段

Ø 复制域：复制域的作用在于将某一个或某几个域中的数据复制到另一个域中

Ø 动态域：动态扩充字段时，需要使用动态域。

l 域需要在【索引库/conf/schema.xml】中配置

4.2 配置一般域

修改 solrhome 的 schema.xml 文件 设置业务系统 Field

<field name="item_goodsid" type="long" indexed="true" stored="true"/>
<field name="item_title" type="text_ik" indexed="true" stored="true"/>
<field name="item_price" type="double" indexed="true" stored="true"/>
<field name="item_image" type="string" indexed="false" stored="true" />
<field name="item_category" type="string" indexed="true" stored="true" />
<field name="item_seller" type="text_ik" indexed="true" stored="true" />
<field name="item_brand" type="string" indexed="true" stored="true" />

4.3 配置复制域

复制域的作用在于将某一个 Field中的数据复制到另一个域中 

<field  name="item_keywords"  type="text_ik"  indexed="true"  stored="false"
multiValued="true"/>
<copyField source="item_title" dest="item_keywords"/>
<copyField source="item_category" dest="item_keywords"/>
<copyField source="item_seller" dest="item_keywords"/>
<copyField source="item_brand" dest="item_keywords"/>

4.4 配置动态域

当我们需要动态扩充字段时，我们需要使用动态域，对于不同情况时是不同的，所以我们需要使用动态域来实现。需要实现的效果如下：

<dynamicField name="item_spec_*" type="string" indexed="true" stored="true" />

5.2.4 增删改查的基本操作
package com.pinyougou.test.solr;

import com.pinyougou.pojo.TbItem;
import org.junit.Test;
import org.junit.runner.RunWith;
import org.springframework.data.solr.core.SolrTemplate;
import org.springframework.data.solr.core.query.Criteria;
import org.springframework.data.solr.core.query.Query;
import org.springframework.data.solr.core.query.SimpleQuery;
import org.springframework.data.solr.core.query.result.ScoredPage;
import org.springframework.test.context.ContextConfiguration;
import org.springframework.test.context.junit4.SpringJUnit4ClassRunner;

import javax.annotation.Resource;
import java.math.BigDecimal;
import java.util.ArrayList;
import java.util.List;
//需要注意两个事项 ：
    //                1 . 查询的时候不需要提交事务，其他的时候都是需要提交事务的
    //                2 . 注意多个对象操作时调用的是 savebeans  单个调用时是 savebean
@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration("classpath:spring/applicationContext-solr.xml")
public class SolrTest {
    @Resource
    private SolrTemplate solrTemplate;
    // 添加
    @Test
    public void testAdd() {
        // 创建实体类
        TbItem item = new TbItem();
        //进行赋值
        item.setId(1L);
        item.setCategory("手机");
        item.setTitle("华为 Mate20Pro 8G 256G");
        item.setSeller("华为官方旗舰店");
        item.setBrand("华为");
        item.setPrice(new BigDecimal(10));
        //进行添加
        solrTemplate.saveBean(item);
        //提交事务
        solrTemplate.commit();
    }
    // 获取
    @Test
    public void testFindOne() {
        //获取 id 为 1 的值，对象取值是 tbitem
        TbItem item = solrTemplate.getById(1, TbItem.class);
        System.out.println(item.getTitle());
    }
    // 删除
    @Test
    public void testDelete() {
        // 根据 id 为 1 的删除
        solrTemplate.deleteById("1");
        // 提交事务
        solrTemplate.commit();
    }
    // 添加集合
    @Test
    public void testAddList() {
        // 添加集合，遍历当前添加对象，遍历 100 次
        List<TbItem> list = new ArrayList();
        for (int i = 0; i < 100; i++) {
            TbItem item = new TbItem();
            item.setId(i + 1L);
            item.setCategory("手机");
            item.setTitle("华为 Mate20Pro 8G 256G");
            item.setSeller("华为官方旗舰店");
            item.setBrand("华为");
            item.setPrice(new BigDecimal(2000 + i));
            list.add(item);
        }
        // 添加集合时，使用的是 savebeans  
        solrTemplate.saveBeans(list);
        // 提交事务
        solrTemplate.commit();
    }
    // 分页展示
    @Test
    public void testPageQuery() {
        // 分页操作需要用到  SimpleQuery  加载的是 * ： *  ==== 表示所有的
        Query query = new SimpleQuery("*:*");
        query.setOffset(20);//开始索引（默认 0）
        query.setRows(20);//每页记录数(默认 10)
        ScoredPage<TbItem> page = solrTemplate.queryForPage(query, TbItem.class);
        System.out.println("总记录数：" + page.getTotalElements());
        List<TbItem> list = page.getContent();
        showList(list);
    }

    //显示记录数据
    private void showList(List<TbItem> list) {
        // 循环查询对象的 标题 和 价格
        for (TbItem item : list) {
            System.out.println(item.getTitle() + item.getPrice());
        }
    }
    // 全部删除
    @Test
    public void testDeleteAll(){
        // 获取到 SimpleQuery 对象 ，删除所有。既 *：*
        Query query=new SimpleQuery("*:*");
        solrTemplate.delete(query);
        // 提交事务
        solrTemplate.commit();
    }
}
